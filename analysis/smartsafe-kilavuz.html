03
Kurulum (OS Bazlı)
💻

🪟 Windows

PowerShell
# 1. Python 3.11 kur — "Add Python to PATH" işaretle!
cd C:\SmartSafe
python -m venv venv
venv\Scripts\activate
pip install -r core\requirements.txt
copy .env.example .env && notepad .env
python core\run.py   # → http://localhost:5000

# Production (Waitress)
pip install waitress
python -c "from waitress import serve; from core.app import create_app; serve(create_app(), host='0.0.0.0', port=5000, threads=8)"

# Windows Servisi (NSSM)
nssm install SmartSafe "C:\SmartSafe\venv\Scripts\python.exe" "C:\SmartSafe\core\run.py"
nssm set SmartSafe AppDirectory "C:\SmartSafe"
nssm start SmartSafe

🐧 Linux (Ubuntu/Debian)

Bash
sudo apt update && sudo apt install python3.11 python3.11-venv python3-pip -y
cd /opt/smartsafe
python3.11 -m venv venv && source venv/bin/activate
pip install -r core/requirements.txt
cp .env.example .env && nano .env
python core/run.py

# systemd servisi
sudo tee /etc/systemd/system/smartsafe.service << 'EOF'
[Unit]
Description=SmartSafe PPE Detection
After=network.target
[Service]
User=ubuntu
WorkingDirectory=/opt/smartsafe
ExecStart=/opt/smartsafe/venv/bin/gunicorn -w 2 -b 0.0.0.0:5000 "core.app:create_app()"
Restart=always
EnvironmentFile=/opt/smartsafe/.env
[Install]
WantedBy=multi-user.target
EOF
sudo systemctl enable --now smartsafe

brew install python@3.11

05
Docker ile Deployment (Önerilen)
🐳
Docker kullanıldığında müşteri kaynak kodunu görmez ve kurulum tek komuta iner.

Dockerfile

Dockerfile
FROM python:3.11-slim
RUN apt-get update && apt-get install -y \
    libgl1-mesa-glx libglib2.0-0 libsm6 libxext6 libxrender-dev \
    && rm -rf /var/lib/apt/lists/*
WORKDIR /app
COPY core/requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 5000
HEALTHCHECK --interval=30s --timeout=10s \
    CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:5000/')" || exit 1
CMD ["python", "core/run.py"]

docker-compose.yml

YAML
version: '3.8'
services:
  smartsafe:
    build: .
    ports: ["5000:5000"]
    volumes:
      - ./models:/app/models
      - smartsafe_data:/app/data
      - smartsafe_logs:/app/logs
    env_file: .env
    restart: unless-stopped
    networks: [smartsafe_net]
  nginx:
    image: nginx:alpine
    ports: ["80:80"]
    volumes: ["./nginx.conf:/etc/nginx/conf.d/default.conf:ro"]
    depends_on: [smartsafe]
    restart: unless-stopped
    networks: [smartsafe_net]

Tek Komutla Ayağa Kaldır

Bash
docker-compose up -d
docker-compose ps && docker-compose logs -f smartsafe
# Güncelleme:
docker-compose down && docker-compose build --no-cache && docker-compose up -d
06
Kamera Altyapısı — Tüm Senaryolar
📷
1
DVR/NVR var
⚡ EN SIK
100 kamera, hepsi DVR'a bağlı. Sadece belirli alanlarda PPE istiyor.
⚡ TEK IP → TÜM KANALLAR · DVR IP = 192.168.1.100 → Kanal 1–64 RTSP
2
Her kameranın ayrı IP'si
IP kameralar switch'e direkt bağlı. IT'den IP listesini al → Toplu IP Ekleme kullan.
3
Birden fazla DVR (Büyük Tesisler)
3 bina, her birinde ayrı DVR. Her DVR'ı SmartSafe'e ayrı ayrı ekle.
⚠️ SmartSafe TÜM DVR ağlarına erişebilmeli (VLAN routing!)
4
VLAN ile izole kamera ağı
Kameralar 10.x.x.x, ofis 192.168.x.x. IT'den routing açtır veya sunucuya 2. NIC tak.
5
Uzaktan Erişim (VPN)
SmartSafe merkezi, kameralar müşteri binasında. Site-to-Site VPN kur (WireGuard / IPsec).
⚠️ ~2-5 Mbps / kamera · Latency < 100ms

08
IP Kamera (Bağımsız) Bağlantısı
📡
Tek Kamera: Kamera Yönetimi → "Kamera Ekleme" → IP, port, kullanıcı/şifre → "⚡ Hızlı Test" → Kaydet.

Akıllı Tespit: IP girip "⚡ Akıllı Tespit" → Sistem bilinen portları tarar, marka ve RTSP URL'i otomatik tespit eder.
09
ONVIF Keşfi
🔍
Kamera Yönetimi → "ONVIF Keşif" → Ağ aralığı girin (192.168.1.1 → 192.168.1.254) → "🔍 ONVIF Tarama" → Bulunanları ekleyin.
⚠️ Kurumsal ağlarda ONVIF çoğunlukla çalışmaz: UDP multicast kapalı · VLAN izolasyonu · Bazı markalar varsayılan kapalı

Çözüm: IT'den IP listesini alın → Toplu IP Ekleme kullanın.

14
Ağ Yapılandırması & Firewall
🔧

Port Gereksinimleri

PortProtokolYönAçıklama
5000TCPGelenWeb arayüzü (HTTP)
554TCPGidenRTSP — kameralara bağlanma
80/443TCPGidenONVIF / kamera web arayüzü
3702UDPGelen+GidenONVIF WS-Discovery (multicast)
Windows
netsh advfirewall firewall add rule name="SmartSafe Web" dir=in action=allow protocol=TCP localport=5000
netsh advfirewall firewall add rule name="ONVIF Discovery" dir=in action=allow protocol=UDP localport=3702
Linux
sudo ufw allow 5000/tcp comment "SmartSafe Web"
sudo ufw allow 3702/udp comment "ONVIF Discovery"
Ana stream (1080p): ~4-8 Mbps/kamera  ·  Alt stream (D1): ~1-2 Mbps/kamera
8 kamera × 4 Mbps = 32 Mbps  ·  64 kamera × 2 Mbps (sub) = 128 Mbps (gigabit gerekli)
💡 Detection için sub stream yeterlidir — bant genişliğini %70 azaltır.
